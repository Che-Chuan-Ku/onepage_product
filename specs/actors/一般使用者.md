# 一般使用者

## 描述
後台系統的店家角色（受限權限）。可存取商品管理、網站管理、訂單查詢、庫存管理、Email 模板設定等功能，但所有資料嚴格隔離，僅能查看並操作自己的資料。

## 關鍵屬性
- 透過 Email + 密碼登入後台（REQ-014）
- RBAC 角色：一般使用者（店家）
- MENU 不顯示「使用者管理」（REQ-034）
- 可存取功能：商品管理、網站管理、商品類型、訂單查詢、庫存管理、Email 模板
- 資料隔離原則（REQ-034）：
  - 網站管理：只看到自己建立的網站
  - 商品管理：只看到自己的商品
  - 商品類型：只看到自己設定的類型
  - 訂單查詢：只看到自己網站的訂單
  - Email 模板：只能設定自己的模板
- API 層強制 owner filter，不可透過 API 取得他人資料（REQ-034）
- 帳號可由管理員建立（REQ-033）

## 來源
- REQ-014: 使用者認證
- REQ-033: 由管理員建立使用者
- REQ-034: RBAC 資料隔離（各功能模組僅操作自己的資料）
